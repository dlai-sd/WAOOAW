## Trial Lifecycle

### Trial Stages

**Day 0: Trial Activation**
- Trial starts after customer configures credentials
- 7-day countdown begins (168 hours)
- Welcome email sent with trial overview
- First goal executed (guided during onboarding)
- Deliverable #1 created and approved

**Day 1-2: Early Engagement**
- Customer exploring platform features
- Goals executing automatically
- Deliverables accumulating in approval queue
- Customer success manager (CSM) monitors engagement
- Automated check-in email at 24 hours

**Day 3-4: Mid-Trial Check-in**
- CSM reaches out (email + calendar invite)
- Review customer satisfaction and value perception
- Address any technical issues or questions
- Discuss use case expansion opportunities
- Gauge conversion likelihood

**Day 5: Pre-Conversion Outreach**
- Urgency messaging begins ("2 days left")
- Highlight deliverables created (quantify value)
- Present conversion options and pricing
- Schedule conversion call if not yet scheduled
- Trial status banner shown prominently in CP

**Day 6: Conversion Decision Day**
- Final outreach ("24 hours remaining")
- Conversion offer with incentives (if applicable)
- Technical preparation for conversion
- Address final objections

**Day 7: Trial Expiration**
- Trial expires at exact 168-hour mark
- If converted: Transition to paid seamlessly
- If not converted: Agent deactivated, deliverables archived but accessible
- Post-trial survey sent

**Post-Trial (Day 8-30)**
- Win-back campaign for non-converters
- Continue email nurture sequence
- Offer to extend trial (rare, approval required)

## Technical Conversion Process

### Pre-Conversion Checklist (CSM/Ops)

Before initiating conversion, verify:

- [ ] Customer explicitly agreed to convert (verbal or written confirmation)
- [ ] Payment method on file (credit card, invoice details)
- [ ] Agent fully configured (all credentials valid)
- [ ] At least 1 goal executed successfully
- [ ] No open technical issues blocking agent
- [ ] Customer understands pricing (â‚¹12,000/month standard)
- [ ] Trial status: "active" (not already expired)

## Billing Activation

### Payment Methods Supported

1. **Credit/Debit Card** (Primary)
   - Visa, Mastercard, Rupay
   - Processed via Razorpay
   - Automatic recurring billing

2. **UPI Autopay** (India)
   - Set up via Razorpay
   - Automatic renewals
   - Mandate-based billing

3. **Invoice/Bank Transfer** (Enterprise Only)
   - Net 30 terms
   - Manual approval required
   - Minimum â‚¹1,44,000/year commitment

### Billing Cycle Options

**Monthly** (Standard):
- â‚¹12,000/month
- Billed on same day each month
- Cancel anytime, prorated refund

**Annual** (10% discount):
- â‚¹1,29,600/year (save â‚¹14,400)
- Billed upfront
- Cancel anytime, prorated refund after 3 months

### Failed Payment Handling

**Day 0 (Payment Fails)**:
1. Retry payment automatically (3 attempts over 3 days)
2. Email customer: "Payment failed, please update payment method"
3. Agent continues operating (grace period)

**Day 3 (Still Failed)**:
1. Suspend agent (pause, don't delete)
2. Email customer: "Final notice before suspension"
3. Customer cannot approve new deliverables

**Day 7 (Still Failed)**:
1. Deactivate agent completely
2. Archive deliverables
3. Account marked "suspended"
4. Win-back campaign begins

**Reactivation After Failed Payment**:
```bash
# Customer updates payment method in CP > Billing
# Ops manually reactivates:
curl -X POST -H "Authorization: Bearer $PP_TOKEN" \
  https://api.waooaw.com/pp/subscriptions/<subscription_id>/reactivate \
  --data '{"reason": "Payment updated, customer requested reactivation"}'

# Verify agent resumes
```

## Churn Prevention

### Early Warning Signs (During Trial)

- [ ] Customer hasn't logged in for 2+ days
- [ ] No goals approved (deliverables piling up)
- [ ] Customer service tickets expressing dissatisfaction
- [ ] Low engagement score (< 2 logins/day)
- [ ] Agent configuration incomplete/broken

**Action**: Proactive outreach within 24 hours of warning sign

### Post-Conversion Retention (First 90 Days Critical)

**Day 30 Check-in**:
- Email: "One month in - how's it going?"
- Review usage metrics with customer
- Address any issues
- Upsell opportunities (more agents, annual plan)

**Day 60 Check-in**:
- "Quick feedback  request"
- NPS survey (Net Promoter Score)
- Feature requests
- Renewal confirmation reminder (if annual trial)

**Day 90 Check-in**:
- "Quarterly business review"
- ROI analysis (quantify value delivered)
- Roadmap preview (upcoming features)
- Contract renewal discussion (if applicable)

### Churn Risk Indicators (Paid Customers)

- Payment failures (3+ attempts)
- Declining usage (agent utilization <50% for 2 weeks)
- Support tickets expressing frustration
- Downgrade requests
- Competitor mentions

**Action**: Escalate to account manager, offer retention incentives
